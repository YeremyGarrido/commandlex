## 2. ADRs — Decisiones de Arquitectura

### ADR-001 · Sistema de UI: CodeWiki CSS ✅ CERRADO Y LIMPIO
- **Fecha:** 2026-02-22
- **Decisión:** `globals-codewiki.css` con variables `--cw-*` es el único sistema de estilos.
- **Eliminados:** `MaterialHomePage.tsx`, `MaterialLayout.tsx`, `MaterialCommandCard.tsx`, `MaterialCategoryPicker.tsx`, `HomePage.tsx`, `globals-material.css`, `globals.css`, `migrate-to-material.sh`, `FAQ-MATERIAL.md`
- **Consecuencias:** No usar clases `md-*` ni variables `--md-sys-*` en ningún componente nuevo.

### ADR-002 · Routing: URL params como fuente de verdad
- **Fecha:** anterior a 2026-02-22
- **Decisión:** Estado de búsqueda (`?q=`) y categoría (`?category=`) vive en la URL via `useSearchParams` / `useRouter`.
- **Consecuencias:** Toda lógica de filtrado lee de `searchParams`, no de estado local.

### ADR-003 · Persistencia de favoritos: localStorage
- **Fecha:** anterior a 2026-02-22
- **Decisión:** IDs de favoritos en `localStorage["favs"]` via `src/lib/favs.ts` (Gateway Pattern).
- **Consecuencias:** Sin sincronización entre dispositivos. Limitación conocida, roadmap v4.0.

### ADR-004 · Export estático: GitHub Pages
- **Fecha:** anterior a 2026-02-22
- **Decisión:** `output: export`. Build genera `/out`. Deploy via `peaceiris/actions-gh-pages`.
- **Consecuencias:** Sin Server Actions, sin API Routes, sin ISR.

### ADR-005 · Dataset: JSON local estático
- **Fecha:** anterior a 2026-02-22
- **Decisión:** `public/data/commands.json` importado como módulo TypeScript en build time.
- **Consecuencias:** Añadir comandos requiere editar JSON y hacer deploy.

### ADR-006 · Stack plataforma de estudio (2026-02-28) 🔄 EN CURSO
- **Contexto:** CommandLex evoluciona de catálogo estático a plataforma de estudio con quizzes, flashcards, progreso de usuario y contenido desde repos de GitHub.
- **Decisiones:**
  - **Deploy:** Cloudflare Pages — reemplaza GitHub Pages. Gratis, global, sin cold starts, soporta Next.js via `@cloudflare/next-on-pages`.
  - **Base de datos + Auth:** Supabase — PostgreSQL hosted, GitHub OAuth incluido.
  - **Contenido externo:** GitHub REST API v3 — el usuario vincula repos públicos; se leen READMEs y archivos `.md`.
  - **Quizzes/Flashcards:** Generación automática desde JSON existente + markdown parseado de repos. Lógica local, sin IA externa.
  - **Progreso:** Tabla `progress` en Supabase `(user_id, item_id, type, score, completed_at)`.
- **Consecuencias:**
  - Eliminar `output: export` de `next.config.js`.
  - Añadir `@supabase/supabase-js` y `@supabase/auth-helpers-nextjs`.
  - Variables de entorno: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
  - GitHub Pages → Cloudflare Pages.
  - ADR-004 queda obsoleto.
- **Fases:**
  1. ✅ Decidido — Infraestructura base (Supabase + Cloudflare + GitHub OAuth)
  2. ⏳ GitHub como fuente de contenido (leer repos/READMEs/docs)
  3. ⏳ Motor de estudio (flashcards, quizzes, repetición espaciada)
  4. ⏳ Workspaces y dashboard de progreso

## 6. Contratos de Datos

### Tipo `Command` actual (pendiente DT-009)
```typescript
type Command = {
  id: string;
  comando: string;
  entorno?: string;        // legacy v2.x
  aplicaciones: string[];
  nivel: string;           // target: 'básico' | 'intermedio' | 'avanzado' | 'teórico'
  descripcion: string;     // puede contener markdown
  ejemplo: string[];
  requerimientos: string;
  tags: string[];
}
```

### Tipo `Command` objetivo (DT-009 + DT-006)
```typescript
type RenderMode = 'command' | 'snippet' | 'shortcut' | 'status-code';
type Nivel = 'básico' | 'intermedio' | 'avanzado' | 'teórico';

type Command = {
  readonly id: string;
  readonly comando: string;
  readonly aplicaciones: readonly string[];
  readonly nivel: Nivel;
  readonly descripcion: string;
  readonly ejemplo: readonly string[];
  readonly requerimientos: string;
  readonly tags: readonly string[];
  readonly renderMode?: RenderMode;
}
```
